대학원에서 LLM과 멀티모달을 공부하면서, 머릿속에만 두면 금세 흐려지는 것들이 너무 많았습니다. 논문 한 편을 읽고 "아 이렇구나" 했다가, 두 주 뒤에 같은 그래프를 다시 보며 처음 보는 표정을 짓는 일이 반복됐습니다. 그래서 **읽고 만든 것을 글로 남기기로** 했습니다.

## 무엇을 쓸까

거창한 계획은 없습니다. 다만 다음 세 가지는 꾸준히 쌓아가려 합니다.

- **읽은 논문의 핵심 한 장**: 막연한 인상이 아니라, 게재처와 인용수, 코드 공개 여부 같은 객관 지표로 검증한 정리
- **직접 만든 학습 도구의 회고**: 무엇이 동작했고 무엇이 깨졌는지
- **졸업논문으로 좁혀가는 과정**: 학교와 모바일의 유휴 컴퓨팅을 모아 LLM을 돌린다는 방향이, 선행연구를 읽으며 어떻게 다듬어지는지

> 트렌드를 좇기보다, 측정 가능한 근거로 판단하려 합니다. 그게 빌더로서 제가 믿는 방식입니다.

## 코드도 가끔

설명을 위해 짧은 코드 조각을 곁들일 때가 있습니다. 예를 들어 토큰 단위로 텍스트를 자르는 일은 생각보다 자주 등장합니다.

```python
def chunk(tokens, size=512, overlap=64):
    step = size - overlap
    return [tokens[i:i + size] for i in range(0, len(tokens), step)]
```

본문에서 `inline code`는 이렇게 보이고, 표 같은 것도 필요하면 씁니다.

| 구분 | 대상 | 검증 기준 |
| --- | --- | --- |
| 논문 | 학회 게재물 | 인용수, 코드 공개 |
| 도구 | 직접 만든 것 | 실제 운영 여부 |
